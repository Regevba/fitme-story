# PRD: Onboarding

> **ID:** 18.11 | **Status:** Shipped | **Priority:** P1
> **Last Updated:** 2026-04-12

---

## Purpose

Guide new users through first-run setup so they understand the product quickly, connect HealthKit with context, set goals, and reach a meaningful first action without friction.

## Business Objective

Onboarding is a retention and activation surface. It reduces first-run confusion, improves permission opt-in quality, and gives the app enough context to personalize training, nutrition, and recovery from day one.

## Target Persona(s)

- **All new users**

## Current Implementation

### Flow (Shipped v2 UX Alignment)
1. **Welcome** — brand introduction and value framing
2. **Goals** — select primary objective and intent
3. **Profile** — lightweight personal/training context
4. **Consent** — privacy framing and clear expectations
5. **HealthKit Permissions** — guided authorization request
6. **First Action** — route into the product with an immediate CTA

### Current State
- The onboarding feature is shipped in the canonical repo and tracked as complete in `.claude/features/onboarding/state.json`
