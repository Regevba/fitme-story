# PRD: AI / Cohort Intelligence

> **ID:** 18.9 | **Status:** Shipped | **Priority:** P1
> **Last Updated:** 2026-04-04

---

## Purpose

Provide personalized fitness recommendations via a three-tier AI pipeline: local rules (always available), cloud cohort analysis (anonymized), and on-device Foundation Models (iOS 26+) — without ever exposing PII.

## Business Objective

AI intelligence is FitMe's second differentiator after privacy. Federated cohort analysis provides population-level insights (e.g., "users with similar profiles who train 4x/week typically eat 1.8g/kg protein") while keeping individual data private. This creates a network effect: more users → better cohort signals → better recommendations.

## Target Persona(s)

- **Primary:** The Data-Driven Optimizer (wants AI-driven guidance)
- **Secondary:** The Consistent Lifter (wants to know "what should I do differently?")

## Current Implementation

### Three-Tier Pipeline
| Tier | Where | Privacy | Confidence | Details |
|------|-------|---------|------------|---------|
| 1. Local Rules | On-device | Full privacy | 0.25 (baseline) | Rule-based signals from LocalUserSnapshot (sleep debt, high frequency, etc.) |
| 2. Cloud Cohort | Railway (FastAPI) | Banded categorical data only | Variable | JWT-authenticated, JWKS validation, k≥50 anonymity floor |
| 3. Foundation Models | On-device (iOS 26+) | Full privacy | ≥0.4 threshold | Apple Intelligence, runs on private full snapshot |

### Privacy Mechanism
- **Banding:** Continuous values discretized into population-safe ranges before transmission
  - Age → "25-34", BMI → "18.5-24.9", Sleep → "6-7 hours"
- **No PII leaves device** — only categorical bands sent to cloud
- **k-anonymity floor:** Cloud requires k=50 minimum before returning cohort signals
- **GDPR Article 5 compliant** — band extraction prevents individual identification

### Data Models
- **AIRecommendation** — segment, signals[], confidence, escalateToLLM, supportingData
- **LocalUserSnapshot** — demographics (age, BMI, gender), training (phase, days/week, goal), nutrition (calories, protein, meals), recovery (sleep, HR, stress), stats (sessions, steps, consistency)
- **4 segments:** training, nutrition, recovery, stats

### Error Handling
- Rate limited / unauthenticated / network error → fall back to local baseline
- Foundation Model confidence < 0.4 → discard, use cloud baseline
- Insufficient data → return local signals only
- Pre-iOS 26 → FallbackFoundationModel (confidence 0)

### AI Engine Backend
- **Stack:** FastAPI (Python) on Railway
- **Auth:** JWT with JWKS validation
- **Endpoint:** Receives banded snapshot, returns cohort signals
- **Privacy:** Receives only categorical bands, never raw values
