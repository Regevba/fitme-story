# PRD: Settings

> **ID:** 18.7 | **Status:** Shipped | **Priority:** P1
> **Last Updated:** 2026-04-04

---

## Purpose

Provide a grouped settings architecture for managing account, security, goals, preferences, training, nutrition, sync, data, and privacy — organized into sectioned cards with eyebrow category labels.

## Business Objective

Settings support user control and trust — critical for a health data app. Key settings (analytics toggle, data export, account deletion) are GDPR compliance requirements. Well-organized settings reduce support burden.

## Target Persona(s)

- **All personas** — settings is a utility feature

## Current Implementation

### Settings Categories
| Category | Section | Settings |
|----------|---------|----------|
| Account | Account Identity | Display name, email, phone, sign-in provider badge |
| Security | Access on Reopen | Toggle: require biometric unlock on reopen |
| Security | Protection Summary | Encryption status (AES-256 + ChaCha20 layers) |
| GDPR | Account | Delete account (AccountDeletionService, 30-day grace period) |
| GDPR | Data Portability | Export full training history as JSON |
| Devices | Connection Status | Apple Watch pairing status |
| Devices | Actions | Watch connectivity diagnostics / re-pair |
| Goals | Profile Snapshot | Current weight, body fat %, program phase |
| Goals | Body Goals | Target weight, target body fat % |
| Preferences | Units | Metric (kg, cm, km) vs Imperial (lbs, in, mi) |
| Preferences | Appearance | System (auto), Light, Dark |
| Preferences | Stats Carousel | Configure home screen metric tiles |
| Nutrition | Strategy | Goal mode (maintenance, cut, bulk) |
| Training | Cardio Targets | Zone 2 HR range, weekly duration goal |
| Training | Readiness Thresholds | HRV ready threshold (ms), resting HR threshold (bpm) |
| Sync | Status | CloudKit/Supabase status, last sync timestamp |
| Data | Local Storage | Encrypted on-device storage size, sync logs |
| Privacy | Analytics | Toggle: allow anonymous usage analytics |
| Data | Danger Zone | Sign out, delete all local data, full account reset |

### Data Model
- **AppSettings** — @MainActor observable, persisted to UserDefaults (non-sensitive)
  - `unitSystem: UnitSystem` (metric/imperial with conversion helpers)
  - `appearance: AppAppearance` (system/light/dark)
  - `requireBiometricUnlockOnReopen: Bool`

### Architecture
- Single `SettingsView` (~1,074 lines) with sectioned `SettingsSectionCard` components
- Pickers, toggles, navigation links to detail editors
- Propagates via @EnvironmentObject (AppSettings, AppTheme)

## Kill Criteria

See "Kill criteria" row in Success Metrics above. Concrete threshold (added 2026-04-26 per CLAUDE.md rule #2): settings-driven crashes >0.5% sustained for 7 days OR support tickets referencing settings confusion >5/week sustained for 4 weeks → settings surface is considered failed and the section grouping is rebuilt. Tagged T2 — Declared.
