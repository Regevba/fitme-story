# PRD: Authentication & Account

> **ID:** 18.6 | **Status:** In Progress | **Priority:** P0
> **Last Updated:** 2026-04-12

---

## Purpose

Provide secure, frictionless authentication with Apple-first identity management, supporting Apple Sign In (via Supabase OAuth), passkeys (WebAuthn with platform + hardware key support), email/OTP, and biometric app lock — with zero-password architecture.

## Business Objective

Authentication is the gateway to the app. Friction here causes abandonment. Apple Sign In reduces signup to one tap. Passkeys eliminate password-related support burden. Biometric lock protects sensitive health data. Session persistence via Keychain ensures seamless re-entry.

## Target Persona(s)

- **All personas** — every user must authenticate

## Current Implementation

### Auth Methods
| Method | Status | Details |
|--------|--------|---------|
| Apple Sign In | Shipped | Primary flow via SupabaseAppleAuthProvider — OAuth nonce exchange, idToken → Supabase JWT |
| Passkeys (WebAuthn) | Shipped | Platform (Face ID, Touch ID, iCloud Keychain) + hardware security keys (YubiKey) |
| Email/OTP | Shipped when Supabase is configured | Real Supabase-backed registration, verification, login, and password reset are active when runtime credentials are present; DEBUG builds keep a local fallback for simulator/internal workflows |
| Google | Repo-wired, config-gated | Package, target, and source scaffolding are now in the project, and a compile-only iOS simulator build succeeded on 2026-04-12 with `GoogleSignIn` resolved. The UI stays hidden until valid Google client IDs and URL scheme values are configured locally and verified end to end |
| Biometric Lock | Shipped | Face ID / Touch ID via LAContext for app re-entry |

### Data Models
- **UserSession** — provider, userID, displayName, email, phone, avatarURL, sessionToken (passkey signature), backendAccessToken (Supabase JWT), credentialID (WebAuthn), signedInAt
- **AuthProvider** enum — apple, google, facebook (unused), passkey, email
- **PendingEmailRegistration** — in-memory during OTP flow (name, birthday, email, password)
- **EmailRegistrationChallenge** — email, expectedCode, expiresAt

### Architecture
- **SignInService** — handles all auth flows via adapter pattern
- **AuthManager** — state machine managing auth lifecycle + biometric lock
- **SupabaseAppleAuthProvider** — Apple OAuth → Supabase JWT exchange
- **Session persistence** — encrypted JSON blob in Keychain (`com.fittracker.regev`)
- **Three-way session merge** — live Supabase JWT refresh + local metadata + Keychain
- **Biometric lock** — cached LAContext allows crypto ops in same session; auto-lock on background
- **Simulator auto-login** — DEBUG mode + App Review environment variable
