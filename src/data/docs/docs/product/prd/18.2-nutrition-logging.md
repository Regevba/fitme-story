# PRD: Nutrition Logging

> **ID:** 18.2 | **Status:** Shipped | **Priority:** P0
> **Last Updated:** 2026-04-04

---

## Purpose

Enable users to track daily macro intake (protein, carbs, fat), log meals from multiple sources, monitor supplement adherence, and track hydration with adaptive targets based on training day.

## Business Objective

Second-highest engagement driver after training. Users who log meals AND train have the highest retention (cross-feature WAU = North Star metric). Nutrition data feeds the AI recommendation engine for cohort analysis.

## Target Persona(s)

- **Primary:** The Consistent Lifter (tracks macros for body composition)
- **Secondary:** The Data-Driven Optimizer (monitors every nutritional variable)

## Current Implementation

### Screens
| Screen | Description |
|--------|-------------|
| NutritionView | Main dashboard — date nav, macro targets, chronological meal feed, supplements, hydration |
| MealEntrySheet | 4-tab meal logging: Smart (camera/OCR), Manual, Template, Search/Barcode |
| MealSectionView | Individual meal card display (up to 4+ slots) |
| MacroTargetBar | Stacked horizontal progress bar (protein/carbs/fat/remaining) |
| SupplementStackCard | Expandable AM/PM supplement stacks with individual checkboxes |

### Data Models
- **NutritionLog** — meals: [MealEntry], totalCalories, totalProteinG, totalCarbsG, totalFatG, waterML, alluloseTaken
- **MealEntry** — mealNumber, name, calories, proteinG, carbsG, fatG, servingGrams, source (manual/template/search/barcode/photoLabel), eatenAt, status
- **MealTemplate** — reusable meal definitions (name, calories, macros)
- **SupplementLog** — morningStatus, eveningStatus, morningTime, eveningTime, individualOverrides
- **SupplementDefinition** — name, dose, benefit, notes, timing (morning/preWorkout/withMeal/evening/preBed)
- **FoodProduct** — from Open Food Facts (caloriesPer100g, proteinPer100g, etc.)
- **ParsedNutritionLabel** — OCR output with Hebrew/English language detection

### Key Features
- 4-tab meal entry (Smart OCR, Manual, Template, Search/Barcode)
- Barcode scanning (Vision framework → Open Food Facts API)
- Nutrition label OCR (camera + photo picker, Hebrew + English)
- Macro progress bar with daily targets (training vs rest day)
- Supplement AM/PM stacks with individual checkbox tracking
- Supplement streak detection with milestone celebrations (7/14/30/60/90 days)
- Hydration tracking with quick-add buttons (+250/500/1000ml)
- Training day targets (3500ml) vs rest day (2800ml)
- Meal source tracking (identifies logging method)
- Quick-log favorites for fast re-entry
- Allulose tracking toggle

## Kill Criteria

See "Kill criteria" row in Success Metrics above. Concrete threshold (added 2026-04-26 per CLAUDE.md rule #2): <15% of WAU log any meal for 30 consecutive days OR meal-entry abandonment rate >70% sustained for 14 days → feature is considered failed for adoption purposes and meal-entry enhancements are de-prioritized while the entry sheet is rebuilt. Tagged T2 — Declared.
