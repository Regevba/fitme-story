# FitTracker UX Copy Guidelines

## Status

Experimental. This is a lightweight UI writing layer for product work, not a full brand-writing handbook.

## Tone

- calm
- direct
- secure
- supportive
- non-hype

## Headers

- Keep them short and concrete.
- Use one clear idea per header.
- Avoid vague motivational language when the interface can name the task directly.

## CTA labels

- Prefer verb-led labels.
- Keep them predictable and short.
- Good examples:
  - `Log Metrics`
  - `Quick Protein`
  - `Start Rest`
  - `Sign in with Apple`

## Helper copy

- Explain one thing at a time.
- Prefer next-step clarity or reassurance.
- Avoid stacking multiple explanations in a dense card.

## Empty and missing-data states

- Say what is missing.
- Say what the user can do next.
- Keep the next action obvious.

## Warning and error copy

- State the issue.
- State the consequence when relevant.
- State the next action.

## Avoid

- filler encouragement that does not help the task
- repeated reassurance copy in the same surface
- vague action labels
- long marketing-like sentences inside utility screens

## FitTracker-specific notes

- Privacy and auth copy should feel explicit and trustworthy.
- Health and metric surfaces should prioritize clarity over flourish.
- Settings copy should stay category-first and decision-friendly.
