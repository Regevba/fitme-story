## Important Issues

### I1 — Sheet Corner Radius: Wrong token (3 files)

`.presentationCornerRadius(24)` should be `AppSheet.standardCornerRadius` (= `AppRadius.sheet` = 32).

| File | Line | Fix |
|---|---|---|
| `MainScreenView.swift` | 177 | `.presentationCornerRadius(AppSheet.standardCornerRadius)` |
| `NutritionView.swift` | 66 | `.presentationCornerRadius(AppSheet.standardCornerRadius)` |
| `TrainingPlanView.swift` | 103 | `.presentationCornerRadius(AppSheet.standardCornerRadius)` |

### I2 — Touch Targets Below 44pt Minimum

| File | Location | Current size | Fix |

### I3 — Hardcoded Corner Radii in AuthHubView

All button styles in `AuthHubView.swift` use hardcoded `cornerRadius` values bypassing `AppRadius`:

| Component | Hardcoded | Correct token |
|---|---|---|
| `AuthPrimaryButtonStyle` | `cornerRadius: 20` | `AppRadius.medium` (16) |
| `AuthSecondaryButtonStyle` | `cornerRadius: 18` | `AppRadius.medium` (16) |
| `AuthCardButtonStyle` | `cornerRadius: 16` | `AppRadius.medium` (16) |
| `OTPCodeEntryField` | `cornerRadius: 14` | `AppRadius.small` (12) |
| `AuthBannerView` | `cornerRadius: 16` | `AppRadius.medium` (16) |

## Minor Issues (queued for Task 4 — Spacing Migration)

### M1 — Hardcoded Padding Values (15+ files)

Top offenders — values NOT on the 4pt grid:

| Value | Occurrences | Nearest 4pt token |
|---|---|---|
| `14` | 15+ | `AppSpacing.medium` (20) or `AppSpacing.xSmall` (12) |
| `18` | 10+ | `AppSpacing.large` (24) |
| `10` | 8+ | `AppSpacing.xxSmall` (8) |
| `6` | 6+ | `AppSpacing.xxxSmall` (4) |

### M2 — Hardcoded `cornerRadius` in View Bodies (widespread)

Raw `.cornerRadius(N)` / `RoundedRectangle(cornerRadius: N)` across ~20 view files. All to be replaced in Task 4 with `AppRadius.*` tokens.

### M3 — Spacing Value 14 (off-grid)

`padding(14)` is off the strict 4pt grid. Should be `AppSpacing.xSmall` (12) or `AppSpacing.small` (16) depending on context.
