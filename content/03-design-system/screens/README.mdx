## Design System at a Glance

- **~125 semantic tokens** — colors, typography, spacing, shadows, radii (never raw literals)
- **13 reusable components** — buttons, cards, tiles, progress indicators, section headers
- **6/6 screens v2-aligned** — 100% token compliance across all major surfaces
- **13 UX foundations principles** — applied systematically to every screen, documented in [ux-foundations.md](../ux-foundations.md)

## What "v2-Aligned" Means

Every major screen went through a structured refactor:

1. **Audit** the existing screen against the 13 UX principles
2. **Document** every gap (token violations, accessibility issues, missing analytics)
3. **Rebuild** from the design system foundations — bottom-up, not patch-in-place
4. **Verify** against a checklist covering tokens, components, state coverage, accessibility, motion, and analytics

The result: screens that look intentional because they were built from shared foundations, not individually styled.
