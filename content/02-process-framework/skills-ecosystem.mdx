# The Skills Ecosystem -- 11 Specialized AI Agents

## Context

FitMe's development is managed by 11 AI agents, each owning a distinct domain of the product lifecycle. The original system was a single monolithic workflow that handled everything inline. As the product grew, that approach couldn't scale: adding a new domain meant bloating one giant prompt, and you couldn't run a design audit without triggering the entire PM pipeline. The ecosystem replaces that monolith with a hub-and-spoke architecture where every agent works independently AND fits into the orchestrated lifecycle.

### Key Connections

- **/ux --> /design** -- The UX spec is the handoff. /ux owns what and why; /design owns how it looks.
- **/research --> /ux** -- Competitive UX patterns and platform guidelines flow from research into UX planning.
- **/design --> /dev** -- The design token inventory is the contract between designed components and coded components.
- **/analytics --> /qa** -- Analytics tests go in the same test suite as functional tests; one CI gate validates both.
- **/cx --> back to /pm-workflow** -- Post-deployment feedback closes the loop. Root cause classification dispatches corrective work to the responsible agent.

## Rules That Apply to Every Agent

1. **Every agent is standalone.** You can invoke any agent directly without running a full PM workflow.
2. **Every agent is composable.** The hub can chain agents together for a full lifecycle.
3. **Every agent writes to at most one shared file.** Reads from many.
4. **No agent auto-advances.** User approves every phase transition.
5. **Change broadcasts.** When a work item completes, the hub writes to the audit log and notifies downstream agents.
6. **Every agent follows the 5-phase internal lifecycle.** Health Check --> Cache Check --> Research (if miss) --> Execute --> Learn.
7. **Every agent declares its external data sources.** Local adapters, external connectors, shared layer targets, and validation gate behavior are documented.
8. **Every agent has a domain-specific research scope.** When cache misses, the agent knows exactly what to investigate.
9. **Data flows reactively.** Any entry point, any time. A single agent invocation can ripple through the entire shared layer.

## Key Takeaways

- **The monolith-to-microservices pattern applies to AI workflows.** Breaking one giant prompt into 11 specialized agents made each domain deeper while keeping the orchestration layer thin.
- **Shared data is the glue.** Agents don't call each other -- they communicate through 15 JSON files that any agent can read and any debugging session can inspect.
- **Standalone + composable is non-negotiable.** Every agent must work independently (for ad-hoc tasks) and compose into the hub (for lifecycle work). This dual-use design was validated across 20+ production features.
- **The learning cache is the multiplier.** Without it, agents are stateless functions. With it, the ecosystem gets measurably faster on every similar task.
